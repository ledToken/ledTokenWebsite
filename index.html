Energy As A Service, Decentralized

LED EaaS will use machine learning AI to understand user needs and recommend or deploy green technologies, as well as establish connections with miners. Because LED EaaS uses blockchain technology, there will be no scraping of personally identifiable information and no reselling of personal data by LED nor by any third party.

Today’s pre-blockchain platforms routinely invade an end user’s privacy with data scrapes and data reselling. Similarly, LED averts the risk of having one’s credit card information being hacked or tampered with because the platform allows for trans- actions to occur through cryptocurrency and blockchain technology. Such is not the case with existing platforms.

Automated Transaction Platform, Customized

LED comprises three different systems integrated to provide an automated end-to-end solution interacting with the Graphene blockchain: a native application with End to End Encryption Messaging (E2EE), a Transaction Processing System (TPS), and an Automated Smart Contract Design.

The purpose of LED is to provide an infrastructure, and allow organizations to control their own data. Practically speaking, this means each channel will point to its own messaging queue, and the queue can be hosted by the owner of the channel.

The LED Protocol

LED is a blockchain-leveraged, secure mobile protocol designed to increase efficiency with respect to our planet’s electric power, enhanced by a token-based rewards ecosystem. LED provides users access to state of the art green technologies and then rewards those users for taking part in, improving and governing a global electricity sharing ecosystem.

Decentralized Channels

Since all messages are end-to-end encrypted, a channel’s data can be considered private and secure within the channel’s queue and on the user’s app. Only the users with access to the channel can decrypt the messages. This allows orga- nizations with strict data security policies or regulatory compliance requirements to confidently follow those policies without depending on third party providers.

User to Miner Chat

One-on-one chat between users and miners will initially be maintained by a Mosiohosted message queue. All messages will be encrypted on the app, so Mosio will not be able to read or alter the contents of the messages.

Overcoming Low Latency

Our architecture can perform transactions without being subjected to any geographical constraints while at the same time allowing instantaneous transactions to take place on its application with low latency and high volumes.

Transaction Processing System

The Transaction Processing System (TPS) is designed to handle events related to all transactions (Token Distribution Events, Licenses, etc.) on the platform. The system is built to optimize concurrency in its distributed client/ server system.

Mobile

A Mobile Database System (MDS) in a distributed client/server system enables processing of all transactional activi- ties to take place in different modes when the client is connected, disconnected or intermittently connected.
